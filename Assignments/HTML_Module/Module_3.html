1. What are the new tags added in HTML5?

HTML5 introduced several new elements and attributes to enhance the structure and semantics of web documents. Some of the notable new tags added in HTML5 include:

  • <header>: Represents introductory content at the beginning of a section or page.
  • <nav>: Defines a section containing navigation links.
  • <section>: Represents a generic section of a document or application.
  • <article>: Defines an independent piece of content that can stand alone.
  • <aside>: Represents content that is tangentially related to the content around it.
  • <footer>: Defines a footer for a section or page.
  • <figure>: Represents self-contained content, such as images, diagrams, photos, code listings, etc.
  • <figcaption>: Provides a caption or legend for a figure element.
  • <main>: Specifies the main content of a document or application.
  • <video>: Embeds a video into an HTML document.
  • <audio>: Embeds audio content into an HTML document.
  • <canvas>: Provides a space on which you can draw graphics and animations via scripting (usually JavaScript).
  • <progress>: Represents the progress of a task, typically displayed as a progress bar.
  • <meter>: Represents a scalar measurement within a known range.
  • <datalist>: Contains a set of <option> elements that represent the values available for other controls.
  • <details>: Represents a disclosure widget from which the user can obtain additional information or controls.
  • <summary>: Defines a summary, caption, or legend for a <details> element.
  • <time>: Represents a specific period in time or a range of time.
  • <dialog>: Defines a dialog box or window.

2. How to embed audio and video in a webpage?

To embed audio and video in a webpage using HTML5, you can use the

Embedding Audio:

<audio controls>
    <source src="audio-file.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>
  • The controls attribute adds playback controls to the audio player.
  • The <source> element specifies the audio file's source and its MIME type. You can include multiple <source> elements for different audio formats to ensure compatibility with various browsers.

Embedding Video:

        <video controls width="640" height="360">
            <source src="video-file.mp4" type="video/mp4">
            Your browser does not support the video element.
          </video>
  • The controls attribute adds playback controls to the video player.
  • The width and height attributes specify the dimensions of the video player.
  • Similar to the audio element, the element within the video element specifies the video file's source and its MIME type. You can include multiple elements for different video formats.

4. Canvas and SVG tags.

The <canvas> and <svg> tags are both used for creating graphics and visual content in HTML5, but they have different underlying technologies and use cases:

<canvas> Element:

The <canvas> element provides a drawing surface that you can use with JavaScript to dynamically generate and manipulate graphics, such as drawings, animations, and interactive content. It allows for pixel-based rendering and is well-suited for tasks that require real-time rendering or complex animations.

  • Example: <canvas id="myCanvas" width="400" height="200"></canvas>

<svg> Element:

The <svg> (Scalable Vector Graphics) element is a XML-based markup language for describing vector graphics. It's ideal for creating scalable, resolution-independent graphics, such as icons, logos, charts, and illustrations. Unlike the <canvas>, SVG elements are part of the DOM (Document Object Model), making them easier to manipulate with CSS and JavaScript.

  • Example :<svg width="400" height="200"> <circle cx="100" cy="100" r="50" fill="red" /> </svg>
